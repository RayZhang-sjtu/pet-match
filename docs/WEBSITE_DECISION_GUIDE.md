# 网站上线前，你需要决定什么？

这份说明写给不需要亲自编程的项目所有者。你不必一次理解所有技术细节，只需要明确产品希望“长什么样、给谁用、放在哪里、以后准备怎么发展”。

当前项目已经有一个可以继续开发的基础版本，但还没有正式公开上线。下面的决定会影响设计、内容、成本、法律风险和未来维护方式，因此需要由你确认。

## 1. 品牌是什么？

品牌就是用户看到这个网站时形成的整体印象，不只是 Logo。

通常包括：

- 网站正式名称，例如“Pet Match”“宠友人格局”或其他名称；
- 一句话介绍，让用户立刻明白网站做什么；
- 说话语气，例如可爱活泼、温柔治愈、理性可信；
- 主色、字体、插画或照片风格；
- Logo、图标和分享链接时显示的封面图；
- 是否强调“娱乐测试”“科学生活方式评估”或“领养前参考”。

为什么现在要确认：品牌会影响首页、题目措辞、结果页、域名和传播方式。名称以后可以修改，但网站公开传播后再改会增加成本。

当前建议：使用年轻、温柔但不过度幼稚的风格；明确写出“娱乐和初步生活方式匹配”，避免让用户误以为这是专业心理诊断。

## 2. 目标地区是什么？

目标地区指网站主要服务哪个国家或地区的用户。例如中国大陆、港澳台、新加坡，或全球中文用户。

地区会影响：

- 哪些宠物可以合法饲养；
- 常见宠物、品种和领养渠道；
- 兽医服务与养宠成本；
- 未成年人和个人信息相关法律；
- 网站访问速度、服务器位置和备案要求；
- 简体中文、繁体中文、货币及计量单位。

当前建议：先明确一个首发地区，不要在第一版同时覆盖全球。若以中国大陆为主要地区，部署地点、域名备案和部分境外服务的访问稳定性需要单独评估。

## 3. 托管是什么？

托管可以理解为“把网站文件放在一台全天在线、任何人都能访问的服务器上”。现在项目文件只在本地工作区里，托管后才会得到一个公开网址。

常见托管方式包括 Cloudflare Pages、GitHub Pages、Vercel、Netlify，以及中国大陆的云服务商。它们的区别主要是：

- 是否免费，免费额度是否够用；
- 在目标地区的访问速度；
- 是否需要实名认证或备案；
- 发布新版本是否方便；
- 能否配置安全策略和自定义域名；
- 将来加入数据库、账户或后台时是否容易扩展。

当前建议：第一版是无账户、无数据库的静态型网站，托管成本可以很低。先根据目标地区选平台，再确认是否需要付费。未经你确认，项目不会开通付费服务或外部账户。

## 4. 域名是什么？

域名是用户记住和输入的网址，例如 `petmatch.example.com`。托管平台通常会先提供一个免费但较长的临时网址；购买自己的域名后，可以使用更短、更正式的地址。

需要决定：

- 是否第一版就购买独立域名；
- 希望使用什么名称；
- 使用 `.com`、`.cn`、`.app` 等哪种后缀；
- 谁持有域名账户、续费和管理权限；
- 是否需要中国大陆 ICP 备案。

重要提醒：域名应始终注册在项目所有者可控制的账户中，并开启自动续费和双重验证。不要让临时开发者成为唯一持有人。

当前建议：先用托管平台临时网址进行内部验收，正式公开前再购买和绑定域名，避免名称尚未确认就产生费用。

## 5. 技术栈是什么？

技术栈就是“制作和运行网站所使用的一整套工具与规则”。它类似盖房时选择木结构、钢结构，以及使用哪套施工工具。

本项目当前使用 TypeScript、React 和 Sites 构建工具：

- TypeScript 帮助系统提前发现数据和代码类型错误；
- React 负责页面交互，例如切换题目、保存当前答案和展示结果；
- 构建工具把开发文件整理成可以发布的网站；
- 自动测试负责检查题目数量、评分规则和网站能否正常构建。

“是否长期保留当前技术栈”是在问：以后继续沿用这些工具，还是改成更简单的原生 HTML、CSS 和 JavaScript。

长期保留当前技术栈的好处：

- 适合继续增加复杂交互、多个页面和可复用组件；
- 类型检查和自动化工具较完整；
- 新 Agent 更容易按成熟模式接手。

代价：

- 需要维护一些软件依赖并定期升级；
- 构建过程比单个 HTML 文件复杂；
- 简单小网站会携带一些暂时用不到的模板能力。

改用原生网页的好处是依赖少、结构直观、静态托管简单；代价是项目变大后，更容易出现手写状态管理和规则不一致。

当前建议：至少在第一个公开版本完成前保留当前技术栈。它已经搭建并通过检查，现在改写不会给用户带来明显收益。等产品方向稳定后，再根据实际复杂度决定是否精简。

## 6. 宠物结果覆盖范围是什么？

这是在决定测试可能推荐哪些动物。目前草案包括犬、猫、小型宠物、水族宠物，以及“当前更适合先体验”。

加入鸟类或爬行动物并不只是多写几个结果，还需要考虑噪声、寿命、专门设备、合法性、来源、专科兽医和动物福利，因此需要更多地区调查与专业复核。

当前建议：第一版先保留现有范围。可以允许结果页提示其他方向，但不要在资料不足时直接推荐具体异宠品种。

## 7. 内容专业复核是什么？

目前的 30 道题和评分规则来自公开养宠建议的整理，是可测试的内容草案，不是经过心理测量学验证的量表。

专业复核意味着请兽医、动物行为或动物福利相关人员检查：

- 有没有误导或危险建议；
- 是否把品种性格说得过于绝对；
- 现实硬门槛是否合理；
- 推荐结果是否忽略常见健康、费用或照护问题；
- 未成年人提示是否充分。

当前建议：在公开推广前完成至少一次内容复核。在复核完成前，网站应一直显示“娱乐和初步匹配”“内容草案”的提示。

## 8. 保存、分享和访问统计是什么？

当前网站不保存答案、不上传数据、不记录访问者。用户刷新页面后，答题进度会消失。

以后可能考虑：

- 保存进度：下次回来继续答题；
- 分享结果：生成链接或图片；
- 访问统计：了解有多少人使用、在哪一题退出；
- 错误监控：网站出错时自动通知维护者。

这些功能会增加产品价值，但也可能收集设备信息或用户行为，涉及隐私披露、数据保留、第三方服务和未成年人保护。

当前建议：第一版继续不保存、不追踪。等确实需要数据来改进产品时，再单独确认收集什么、为什么收集、保存多久以及由谁访问。

## 9. 你可以怎样回复我？

不需要使用技术术语。请复制下面的清单，填入你的选择；不确定的项目可以写“按推荐方案”或“暂不决定”。

```text
1. 网站正式名称：
2. 一句话介绍：
3. 品牌感觉（可爱 / 温柔 / 理性 / 潮流 / 其他）：
4. 主要目标地区：
5. 主要语言（简体中文 / 繁体中文 / 其他）：
6. 技术栈：保留当前方案 / 希望精简 / 按推荐方案
7. 宠物范围：保留当前范围 / 增加鸟类 / 增加爬宠 / 其他
8. 品种表述：类型优先、品种仅作示例 / 希望推荐明确品种
9. 第一版是否保存答题进度：
10. 第一版是否需要分享结果：
11. 第一版是否使用访问统计：
12. 托管：先用临时网址 / 已有指定平台 / 暂不决定
13. 域名：暂不购买 / 希望购买 / 已有域名
14. 内容专业复核：上线前安排 / 暂时仅作内部测试
15. 其他希望或限制：
```

收到你的明确回复后，下一步会把决定写入 `docs/DECISIONS.md`，同步需求和任务，再按确认范围继续实现。未确认之前，不会擅自购买服务、注册账户、发布网站或加入数据收集功能。
