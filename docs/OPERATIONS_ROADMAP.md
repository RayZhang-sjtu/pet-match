## 1. 用途与执行原则

本文件是后续运营、获客和商业化的主路线图。新 Agent 应先阅读本文件以及 `README.md`、`docs/PRODUCT_REQUIREMENTS.md`、`docs/ARCHITECTURE.md`、`docs/TASKS.md`、`docs/OPEN_QUESTIONS.md`。

一次只推进一个阶段；每项任务必须有可检查的产出和验收证据。价格、指标与时间均为经营假设，必须根据真实用户和客户反馈修正。不得把浏览、点赞或口头兴趣描述为成交或业务验证。

## 2. 当前基线

### 已完成

- 网站已通过 GitHub Pages 上线，无需登录。
- 30 道题：24 道性格偏好题、6 道必要现实条件题。
- 犬 30、猫 30、小宠 15、水族 15，共 90 个明确品种。
- 结果含匹配原因、现实核对与免责声明；现实条件不足时仍缩小展示性格参考结果。
- 可在设备内生成、保存和分享结果图片；结果页和图片含公开网址二维码。
- 答案仅在浏览器内存计算，不上传、不保存；没有账号、数据库、广告、支付或用户追踪。
- 已有格式、lint、类型、单元测试和构建检查。

### 尚未完成

- 动物福利/兽医专业内容复核。
- 手机、桌面、键盘、二维码和图片分享的完整人工验收。
- 真实用户完成率、保存率、分享率和反馈数据。
- 企业合作页、销售材料、标准报价和案例。
- 商业使用所需的新托管、合同、收款和开票路径。

### 商业化红线

- 当前内容是内部测试草案，不宣称科学人格测评或专业养宠结论。
- GitHub Pages 仅用于当前演示；开始收费或承载商业活动前，必须重新确认平台规则并迁移至允许商业用途的托管。
- 目标人群包含未成年人。完成隐私与合规设计前，不收集姓名、联系方式、答案、位置、IP、设备指纹等个人信息。
- 不出售用户数据，不因赞助而暗中改变匹配结果；赞助和推广必须显著标识。

## 3. 90 天目标

长期方向：把 Pet Match 做成一名独立开发者可稳定交付的“宠物品牌互动测试定制服务”，帮助品牌获得参与、停留、分享和活动反馈，同时保持负责任的用户体验。

90 天成功标准：

1. 300–1,000 次真实测试开始。
2. 至少 20 份有效反馈，其中至少 10 人认为结果“有一定契合感”。
3. 得到开始、完成、保存和分享的初步漏斗数据。
4. 至少 10 位合格企业联系人有效回复，完成 3 次需求沟通。
5. 获得 1 个付费试点；若尚未成交，可先完成 1 个允许公开展示数据和评价的公益案例，但不得将其当作商业成交。

首轮内部目标（不是行业基准）：完成率大于 50%，保存或分享率大于 8%，至少 3 位小型创作者自愿发布体验。

## 4. 商业模式与客户

### 第一优先：标准化 B2B 定制服务

用户提出的“C2B”方向，实际成交关系更接近 B2B 产品化服务。企业购买的不是一套题，而是用于新品推广、社交传播、线下活动和私域互动的营销工具。

统一价值表述：

> Pet Match 是可品牌定制的宠物互动测试。用户完成后获得品牌化结果图片并主动分享，适合新品推广、线下活动和社交媒体传播，可按品牌配色、产品和活动目标定制。

优先客户：成长型宠物食品/用品/智能设备品牌；宠物营销与活动代理商；连锁宠物店、医院、商场和展会；可提供公信力案例的救助或领养机构。首轮不优先大型集团，因为采购、法务、开票和决策周期通常更长。

后续才评估：品牌赞助、用品联盟分成、高级个人报告、本地服务线索、周边、用户赞助和展示广告。当前不实施个人数据线索业务和广告；算法推荐必须与商业关系分离。

## 5. 套餐与价格假设

| 套餐       |          建议价格 | 范围                                             | 适用场景                 |
| ---------- | ----------------: | ------------------------------------------------ | ------------------------ |
| 试点版     |      ¥2,999–4,999 | Logo、配色、欢迎页、活动文案、二维码、基础结果页 | 小品牌首次活动           |
| 活动版     |     ¥8,000–20,000 | 定制题目和结果图、匿名报告、活动期支持           | 新品、展会、营销活动     |
| 年度白标版 | ¥20,000–60,000/年 | 独立品牌、多次活动、约定维护支持                 | 代理商、连锁、持续运营方 |

以上不是固定承诺。首批最多两个低价试点；免费仅限一页静态效果示意。默认 7–14 天交付、两轮文案修改；新增功能、第三方接入、个人信息收集和长期运维另行评估。合同应争取匿名数据和案例展示授权。

## 6. 数据漏斗与口径

用户漏斗：`页面访问 → 开始测试 → 完成测试 → 保存/分享 → 反馈`。  
销售漏斗：`企业名单 → 首次联系 → 有效回复 → 需求沟通 → 示意 → 报价 → 试点 → 案例/续费`。

建议匿名事件：`page_view`、`test_started`、`test_completed`、`result_saved`、`result_shared`。只记录日期、事件类型和网站版本的汇总值，不记录答案、IP、指纹、姓名、联系方式或位置。完成率为 `test_completed / test_started`；保存分享率为 `(result_saved + result_shared) / test_completed`。

GitHub 仓库 Traffic 不能代替网站转化漏斗。统计属于新增外部服务和产品行为，必须由所有者确认供应商、成本、数据字段及保留期后再实现。

销售侧每周新增 30 家候选企业，个性化联系 20–30 家，记录回复率、沟通、示意、报价、成交、客单价和交付工时；不制造虚假流量或案例。

## 7. 找企业和表达合作

名单来源：宠物展会展商和协会会员、天猫/京东旗舰店、品牌官网、小红书/抖音/公众号新品信息、营销代理商案例页、脉脉/LinkedIn、公开招聘信息。优先联系创始人、品牌、市场、内容、私域或电商负责人，只使用公开工作联系方式，避免群发骚扰。

线索表字段：企业、品类、官网/店铺、联系人与职务、公开联系方式、来源、个性化观察、首次联系日期、跟进日期、状态、下一步、备注。

首次联系模板：

> 您好，我是 Pet Match 的独立开发者。  
> 我注意到贵品牌最近在推广【具体产品/活动】。这类产品可以配合一个“测出最适合你的宠物伙伴”的互动测试，让用户完成后生成带品牌视觉的结果图片，并在小红书、朋友圈或线下活动中分享。  
> 目前 Pet Match 已有 30 道题、90 个宠物品种结果，支持品牌配色、Logo、活动文案、二维码和分享图片定制。  
> 演示地址：<https://rayzhang-sjtu.github.io/pet-match/>  
> 如果贵品牌近期有新品推广、用户互动或线下活动计划，我可以先免费做一页定制效果示意，不需要贵方立即决定采购。请问这类合作通常由贵方哪位市场或品牌同事负责？

跟进节奏：第 0 天首次联系；第 3–4 天补一张针对对方场景的示意；第 8–10 天最后礼貌跟进，无回复则暂停。无需强调“没有经验”，但必须如实说明独立开发者和试点阶段。

需求沟通必问：活动目标、主要平台、上线日期和周期、成功指标、是否收集联系方式、谁验收、合同和发票要求。

## 8. 一个人可执行的增长方式

首阶段只选一个主渠道，推荐小红书；同一主题再改编为抖音/视频号短视频或 B 站内容，避免同时经营过多平台。

每周围绕一个主题产出一篇主帖、一个短视频、一张结果卡、一个投票、一则品种解释和一个评论问题。选题包括：“内向一定适合猫吗”“小宠真的省事吗”“高能量主人适合哪类犬”“为什么会得到先体验结果”“90 个品种里谁最像你”。短视频前两秒提出问题，展示 2–3 题和结果，邀请用户测试晒图，不承诺百分百准确。

每周联系约 20 位 1,000–30,000 粉丝、互动真实的宠物创作者，提供定制结果图或“猜结果”玩法，不要求生硬口播。可用“主人测自己”“情侣匹配”“宿舍匹配”“博主猜粉丝结果”等形式。

保留结果图和二维码分享闭环；后续可评估好友对比、情侣匹配和邀请测试，但不强制分享。将 90 个品种逐步转为经过复核的科普页，并建立预算、公寓、首次养宠指南。可与救助机构合作领养准备度公益版本，获取真实反馈和案例授权。

## 9. 90 天执行清单

### 阶段 0：可验证基线（第 1 周）

- [ ] **OPS-001 完整人工验收**：检查手机、桌面、键盘、35 题、普通/安全结果、二维码、保存和分享；用截图或记录关闭 `T-107`、`T-122`。
- [ ] **OPS-002 确认匿名统计方案**：输出事件、字段、保留期、服务商/自建、成本和隐私影响；所有者确认并写入决策。
- [ ] **OPS-003 建立反馈方式**：不强制真实姓名，能回答是否完成、是否契合、哪里困惑、是否愿分享；外部表单须先确认。

门槛：人工验收通过，统计和反馈方案获批；此前不承诺可量化营销效果。

### 阶段 1：包装成可购买服务（第 1–2 周）

- [ ] **OPS-101 企业合作页**：场景、价值、演示、套餐、流程、边界、联系方式。
- [ ] **OPS-102 三个虚构品牌示例**：新品、线下活动、公益领养，显著标注“概念示例”。
- [ ] **OPS-103 一页销售 PDF**：问题、方案、效果、交付范围、试点价格、联系方法。
- [ ] **OPS-104 线索表和案例模板**：案例记录目标、方案、真实数据、评价和限制。
- [ ] **OPS-105 交易条件**：确认个人收款、合同、发票、退款、知识产权和案例授权；税务问题咨询当地专业人员。

门槛：陌生企业能在 3 分钟内看懂买什么、适合什么场景、价格和启动方法；没有虚构客户或业绩。

### 阶段 2：第一批真实用户（第 2–4 周）

- [ ] **OPS-201** 发布 3 组小红书主题内容并复用为短视频/结果卡。
- [ ] **OPS-202** 联系 15–20 位小型宠物创作者。
- [ ] **OPS-203** 邀请至少 50 位目标用户，逐步累计到 300 次测试开始。
- [ ] **OPS-204** 完成 20 份有效反馈，将高频问题转为任务。
- [ ] **OPS-205** 每周复盘来源、开始、完成、保存、分享和反馈。

门槛：有真实数据与可引用匿名评价，修复阻碍完成和分享的严重问题，不只依赖熟人反馈。

### 阶段 3：企业外联（第 3–6 周）

- [ ] **OPS-301** 每周新增 30 家合格企业，先聚焦一个品类。
- [ ] **OPS-302** 每周个性化联系 20–30 家，每条包含近期产品/活动观察。
- [ ] **OPS-303** 至少 3 次需求沟通，统一记录问题和采购障碍。
- [ ] **OPS-304** 为高意向客户做一页示意，不免费完成整个项目。
- [ ] **OPS-305** 发出至少 2 份正式报价，写清范围、两轮修改、工期、付款节点和额外计费。

门槛：10 个有效回复、3 次沟通、2 次报价；按真实异议调整套餐，不用无限增加功能换取兴趣。

### 阶段 4：首个试点（第 5–8 周）

- [ ] **OPS-401** 签署试点范围：目标、页面、内容、数据、工期、验收、费用、案例授权。
- [ ] **OPS-402** 商业版上线前迁移至允许商业用途且成本明确的托管；域名单独决策。
- [ ] **OPS-403** 复用现有架构交付定制，不复制评分规则，客户内容与核心规则分离。
- [ ] **OPS-404** 输出只含真实匿名汇总数据、统计口径和限制的活动报告。
- [ ] **OPS-405** 获取客户评价和案例授权；不同意公开则只做内部复盘。

门槛：1 个付费试点，或 1 个有公开授权的公益案例；工时和成本足以修正报价。

### 阶段 5：可重复业务（第 9–12 周）

- [ ] **OPS-501** 标准化需求表、报价、内容清单、验收、发布、报告和复盘。
- [ ] **OPS-502** 联系 5 家宠物营销代理商，测试转售或联合交付。
- [ ] **OPS-503** 达成第 2–3 个付费试点，验证并非偶然成交。
- [ ] **OPS-504** 按实际工时重定价，淘汰低利润、难维护的定制项。
- [ ] **OPS-505** 多次验证采购需求后，再决定经营主体与开票路径。

门槛：出现重复销售、稳定交付边界和可信案例后，才扩大内容、付费功能或协作人数。

## 11. 风险与待确认决策

| 风险                      | 控制措施                                         |
| ------------------------- | ------------------------------------------------ |
| 未复核内容造成错误建议    | 保留草案声明；商业宣传前复核高风险内容           |
| 未成年人和个人信息        | 当前不收集；任何新增收集先做隐私和法律评估       |
| GitHub Pages 商业用途限制 | 商业版本上线前迁移；当前站点不处理支付和敏感交易 |
| 没有精确数据              | 获批后建立最小匿名事件漏斗                       |
| 个人开发者被无限定制拖垮  | 固定套餐、两轮修改、额外需求另报价               |
| 虚假专业性或业绩          | 指标可追溯；不宣称心理诊断或科学效度             |
| 赞助损害推荐可信度        | 算法与商业关系分离，广告显著标识                 |
| 合同和发票阻碍采购        | 先做小企业试点，以真实需求决定是否设经营主体     |

执行到对应任务时，以下决策必须写入 `docs/OPEN_QUESTIONS.md` 并由所有者确认：统计服务/成本/保留期；反馈表单及联系方式；商业托管与域名；收款、主体和开票；首批最低价格与免费边界；专业复核人资质；赞助、分成和广告标识规则。
