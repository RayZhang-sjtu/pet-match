# 架构决策

## ADR-001：本地无账户单页 MVP（已接受，2026-07-11）

采用浏览器内存计算和静态友好部署；无数据库、账号、追踪或外部 API。隐私和维护风险低，代价是刷新后进度消失。

## ADR-002：内容、评分、界面分层（已接受）

题目资料、纯评分函数和页面流程分别放置，降低内容修改破坏算法的风险。

## ADR-003：可解释规则而非机器学习（已接受）

使用八维加权与现实门槛；同输入有稳定结果。权重仍是待验证草案。

## ADR-004：Sites TypeScript/React 骨架（暂定）

用于类型、构建和托管一致性，无付费服务。原生 HTML 更轻但类型与扩展保障较弱；长期选择见 OQ-001。

## ADR-005：品牌、地区与内容范围（已接受，2026-07-11）

正式名称为 Pet Match，主要面向中国大陆简体中文用户；视觉采用可爱温柔的绿粉色圆角风格。第一版保留犬、猫、小宠、水族和“先体验”，结果给出明确代表品种，但必须同时呈现个体差异和动物福利提示。

## ADR-006：无持久化的结果分享（已接受，2026-07-11）

第一版不保存答题进度、不使用访问统计。分享功能只调用浏览器/手机系统分享，或复制结果文字和当前网页链接；不把答案编码进链接，也不发送到服务器。

## ADR-007：使用 GitHub Pages 临时托管（已接受，2026-07-11）

暂不购买域名，使用 GitHub Pages 提供的项目临时网址。保留现有 Sites 开发构建，同时增加 Next.js 纯静态导出与 GitHub Actions 发布流程。GitHub Pages 不运行服务器逻辑，因此未来加入账户、数据库或服务端 API 时必须重新评估托管方案。内容暂不做专业复核，不得作为正式专业建议推广。
