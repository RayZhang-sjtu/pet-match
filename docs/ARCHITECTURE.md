# 架构

当前采用单页 React + TypeScript 网站，无后端数据、账户或外部服务；答案和评分只存在浏览器内存。

依赖方向：`src/content`（题目与结果）→ `src/domain`（验证、评分、匹配）→ `app`（流程与展示）。业务层是确定性纯函数，不访问 DOM、网络或全局状态；非法输入给出明确错误。内容使用稳定 ID。页面不复制评分规则。

答案 1–5 归一化为十二个生活方式与性格维度。题库由 23 道性格题和 12 道必要现实条件题组成。现实硬门槛不满足时主结果为“先体验”，但同一答案仍会忽略门槛计算一个缩小版品种参考；其他结果计算用户向量与 90 个品种理想画像的平方距离，选择最接近者。并列按稳定 ID 决定。

结果图片由 `src/ui/result-card.ts` 在浏览器 Canvas 中生成，Blob 只存在于当前页面，保存和系统分享均不经过服务器。`qrcode` 在浏览器本地把固定公开网址转换为二维码；网页和分享图片不调用二维码服务。

若出现账户、跨设备保存、动态题库或后台，必须先做新架构与隐私决策，不能把数据访问直接混入页面。
